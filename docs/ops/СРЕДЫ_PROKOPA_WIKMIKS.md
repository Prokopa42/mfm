# МФМ: разделение сред ProKopa и Wikmiks

Дата решения: 2026-06-25

## 1. Смысл решения

МФМ должен жить как один продуктовый проект с двумя отдельными средами:

- `ProKopa` - личная тестовая среда Алексея;
- `Wikmiks` - стабильная утвержденная среда для остальных пользователей.

Это не два разных продукта и не две независимые кодовые базы. Это две раздельные копии одного продукта с разным правилом обновления.

## 2. Факт на момент решения

Перед разделением оба домена обслуживались одним приложением:

```text
/opt/mfm
pm2: mfm
backend: 127.0.0.1:3002
domains:
  https://mfm.prokopa.ru/
  https://mfm.wikmiks.ru/
```

Практический риск: любое обновление `/opt/mfm` сразу меняло и личный тестовый домен, и публичный стабильный домен.

## 3. Целевая серверная схема

```text
/opt/mfm-prokopa
pm2: mfm-prokopa
backend: 127.0.0.1:3003
domain: https://mfm.prokopa.ru/
назначение: личная тестовая среда

/opt/mfm-wikmiks
pm2: mfm-wikmiks
backend: 127.0.0.1:3002
domain: https://mfm.wikmiks.ru/
назначение: стабильная среда для пользователей
```

Старый каталог `/opt/mfm` после миграции не должен быть активным источником приложения. Его можно оставить как архивный ориентир до отдельного решения об очистке.


Текущий путь:

```text

```

используется как текущая исходная рабочая папка до завершения миграции. После миграции нужно явно выбрать его статус: архив, управляющий репозиторий или одна из двух сред.

## 5. Правило продвижения версии

Рабочий поток:

1. Все эксперименты и правки сначала идут в `ProKopa`.
2. `mfm.prokopa.ru` можно обновлять часто и без риска для внешних пользователей.
3. После ручной проверки версия получает статус "утверждена".
4. Только утвержденная версия переносится в `Wikmiks`.
5. `mfm.wikmiks.ru` обновляется отдельной командой, а не автоматически вместе с тестовой средой.

Правило по умолчанию от 2026-06-25:

- если Алексей просит "выгрузи", "задеплой", "опубликуй" или "залей" МФМ без отдельного уточнения среды, Codex должен считать целью `ProKopa`;
- `Wikmiks` нельзя обновлять по умолчанию;
- `Wikmiks` обновляется только если Алексей явно говорит, что нужно выгрузить стабильную/публичную версию на `Wikmiks` или `mfm.wikmiks.ru`.

Запрещено:

- деплоить незавершенную рабочую копию на `Wikmiks`;
- использовать один и тот же PM2-процесс для двух доменов;
- направлять оба домена на один backend-порт;
- хранить пароли, токены и закрытые ключи в Obsidian, README, handoff или git.

## 7. Доступы и секреты

Фиксировать в открытой документации можно:

- домены;
- IP-адреса;
- SSH-пользователя;
- порты;
- PM2-процессы;
- пути на сервере;
- где хранится секрет.

Фиксировать в открытой документации нельзя:

- пароли;
- закрытые SSH-ключи;
- токены;
- cookie;
- резервные коды;
- содержимое `.env`, если там есть секреты.

Текущая рабочая ручка доступа, обнаруженная 2026-06-25:

```text
ssh root@192.168.10.135 -p 22
host: wikmiks.ru
```

Публичные домены на момент проверки указывали на:

```text
mfm.prokopa.ru -> 79.164.244.249
mfm.wikmiks.ru -> 79.164.244.249
```

Секреты должны храниться вне проекта: в менеджере паролей, SSH-agent или системном хранилище ключей. В проекте фиксируется только ссылка-указатель: "секрет хранится вне git".

## 8. Минимальная проверка после разделения

На сервере:

```sh
pm2 status mfm-prokopa
pm2 status mfm-wikmiks
curl -I http://127.0.0.1:3003/
curl -I http://127.0.0.1:3002/
angie -t
```

Снаружи:

```sh
curl -I https://mfm.prokopa.ru/
curl -I https://mfm.wikmiks.ru/
```

Ожидаемый результат:

- оба домена возвращают `200`;
- `mfm.prokopa.ru` и `mfm.wikmiks.ru` имеют разные backend-порты;
- обновление ProKopa не меняет Wikmiks;
- обновление Wikmiks выполняется только отдельным продвижением версии.

## 9. Текущий статус фиксации

Эта памятка создана как проектное решение и локальный источник для дальнейшей миграции.

Статус после применения 2026-06-25:

- `mfm.prokopa.ru` вынесен в отдельную серверную среду:

```text
/opt/mfm-prokopa
pm2: mfm-prokopa
backend: 127.0.0.1:3003
domain: https://mfm.prokopa.ru/
```

- `mfm.wikmiks.ru` сознательно не обновлялся и остался на прежней стабильной среде:

```text
/opt/mfm
pm2: mfm
backend: 127.0.0.1:3002
domain: https://mfm.wikmiks.ru/
```

- Angie изменён только для `mfm.prokopa.ru`: `proxy_pass` переведён с `127.0.0.1:3002` на `127.0.0.1:3003`.
- Angie backup перед правкой:

```text
/etc/angie/backups/mfm_prokopa_split_20260625_185048
```

- Проверки после применения:

```text
angie -t -> successful
pm2: mfm online, mfm-prokopa online
curl -I http://127.0.0.1:3003/ -> 200
curl -I http://127.0.0.1:3002/ -> 200
curl -I https://mfm.prokopa.ru/ -> HTTP/2 200
curl -I https://mfm.wikmiks.ru/ -> HTTP/2 200
```

Оставшаяся миграция:

- отдельный каталог `/opt/mfm-wikmiks` и процесс `pm2 mfm-wikmiks` ещё не создавались;
- продвижение версии в `Wikmiks` по-прежнему требует отдельной явной команды Алексея.
